El css permite animar elementos html sin necesidad de recurrir a JS

Animaciones

Una animacion es el cambio gradual de un estilo a otro se pueden cambiar tanstas propiedades de CSS como quiera tanstas veces como sea necesario

para usar una animacion debe especificar la regla @keyframes

La regla @keyframes

Esta regla contiene los estilos que el elemnto tendra un determinado momnto.

Cuando se especifica una regla los cambios se ejecutaran de manera gradual en ciertos momentos. Para que la animacion funcione,debe asignar un keyframe al css de un elemento. ejemplo: ref al css linea 1

cuando una animacion finaliza, el estado del elemento revierte a su forma oreigianal.

Propiedades de las animaciones

  • animation-delay
  • animation-iteration-count
  • animation-direction
  • animation-timing-function
  • animation-fill-mode

animation-delay:

Especifica el retardo del inicio de una animación.

Los valores positivos indican el tiempo que ha de transcurrir para la animación de comienzo. Los valores negativos indican que la animación empezará como si ya hubiese transcurrido el tiempo especificado.

animation-iteration-count

Establece el número de veces que se repetirá una animación.

Se puede establecer un número natural de veces o se puede indicar un número ilimitado con el valor infinite.

animation-direction

Especifica si la animación se producirá en orden normal, inverso o irá alternando ciclos. Los valores posibles son:

  • normal La animación se reproduce hacia adelante. Valor por defecto.
  • reverse La animación se reproduce hacia atrás (de 100% a 0%).
  • alternate La animación se reproduce hacia adelante, y luego hacia atrás.
  • alternate-reverse La animación se reproduce hacia atrás, y luego hacia adelante.

animation-timing-function

Especifica la curva de velocidad al reproducir una animación.

Puede tener los siguientes valores:

  • ease La animación tendrá un inicio lento, luego irá rápido y tendrá un final lento. Valor por defecto.
  • linear La animación tendrá una velocidad estática durante toda la duración.
  • ease-in La animación tendrá un inicio lento.
  • ease-out La animación tendrá un final lento.
  • ease-in-out La animación tendrá un inicio lento y un final lento.
  • cubic-bezier(n, n, n, n) Permite al usuario especificar la curva de velocidad mediante una función.

animation-fill-mode

por defecto las animaciones no afectan los elementos antes de iniciar o despues de finalizar. Esta propiedad cambia este comportamiento

la propiedad animation-fill-mode establese el estilo del elemento objerivo cuando la animacion no se esta reproducioendo.

los valores posibles:

  • noneValor por defecto,la animacion no aplica cambios de estilo
  • forwardEl elemento retendra el estilo establesido en el ultimo keyframe de la animacion (dependera de si se estareproduciendo normal o manera invertida animation-direction y el numero de interacciones animation-iteration-count).
  • backwardsEl elemento retentra el estilo establecido enelpreimer keyframe de la animacion.dependera de la direccion y tomara el estilo durante el retardo proporcionado por animation-delay
  • bothLa animacion seguira las reglas combinadas de forward y backwards
